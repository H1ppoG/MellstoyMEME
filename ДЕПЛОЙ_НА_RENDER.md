# Как выложить сайт в интернет (Render)

После деплоя у тебя будет постоянная ссылка вида `https://mellstroy-memes.onrender.com` — её можно скинуть друзьям, сайт будет доступен с любого устройства и сети.

---

## 1. Установить Git и залить проект на GitHub

### 1.1 Установить Git

- Скачай Git: **https://git-scm.com/download/win**
- Установи с настройками по умолчанию.

### 1.2 Создать репозиторий на GitHub

1. Зайди на **https://github.com** и войди (или зарегистрируйся).
2. Нажми **"New repository"** (или "+" → New repository).
3. Название, например: **mellstroy-memes**.
4. Оставь **Public**, галочку "Add a README" можно не ставить.
5. Нажми **Create repository**.

### 1.3 Залить папку проекта в репозиторий

В терминале (PowerShell или cmd) перейди в папку проекта:

```bash
cd C:\Users\idontcareaboutsleep\Desktop\MemeSite
```

Выполни по очереди (подставь вместо `ТВОЙ_ЛОГИН` свой логин GitHub):

```bash
git init
git add .
git commit -m "First commit"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/mellstroy-memes.git
git push -u origin main
```

При `git push` GitHub попросит войти (логин и пароль или токен). Если репозиторий только что создан, по инструкциям на экране можно создать токен (Settings → Developer settings → Personal access tokens) и использовать его вместо пароля.

**Важно:** в репозиторий попадут и видео из папки `videos/`. Если там много больших файлов (например, больше 100 МБ на файл), GitHub может отказать. Тогда часть видео можно не добавлять в Git или использовать Git LFS.

## 2. Создать сервис на Render

1. Зайди на **https://render.com** и войди через GitHub (Sign in with GitHub).
2. Нажми **"New +"** → **"Web Service"**.
3. Подключи репозиторий:
   - Если репо уже есть в списке — выбери **mellstroy-memes** (или как назвал).
   - Если нет — нажми **"Connect account"** и дай Render доступ к GitHub, затем выбери репозиторий.
4. Настрой сервис:
   - **Name:** mellstroy-memes (или любое).
   - **Region:** выбери ближайший (например Frankfurt).
   - **Branch:** main.
   - **Runtime:** Node.
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
5. **Environment** (переменные окружения):
   - Нажми **"Add Environment Variable"**.
   - **Key:** `ADMIN_KEY`
   - **Value:** придумай длинный секретный ключ (например случайный пароль) и введи его. Этот ключ нужен для входа в админку на `https://твой-сайт.onrender.com/admin.html`.
6. Нажми **"Create Web Service"**.

Render начнёт сборку и деплой. Подожди 3–5 минут. Вверху появится ссылка вида **https://mellstroy-memes.onrender.com** — это и есть твой сайт.

---

## 3. Проверить и передать ссылку

- Открой в браузере ссылку из Render.
- Админка: **https://твой-сайт.onrender.com/admin.html** — войди с тем ключом, который задал в `ADMIN_KEY`.
- Друзьям скинь просто ссылку на главную (например `https://mellstroy-memes.onrender.com`).

---

## 4. Важно про бесплатный тариф Render

- **Сайт будет доступен всегда** по одной ссылке.
- После ~15 минут без заходов бесплатный сервис "засыпает"; первый заход после этого может открыться с задержкой 30–50 секунд — это нормально.
- **Файлы, которые пользователи загружают** (заявки с видео), на бесплатном тарифе **могут пропадать после перезапуска** сервера (Render использует временный диск). Сами видео из папки `videos/`, которые лежат в репозитории, остаются и отображаются. Если понадобится сохранять заявки и новые видео навсегда, можно потом подключить базу данных или постоянный диск (платные опции).
